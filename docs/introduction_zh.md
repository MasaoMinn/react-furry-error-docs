# 项目介绍

## react-furry-error

一个可爱且实用的 React 项目错误提示工具。
让令人头疼的报错，变成毛茸茸的兽设反应。

## react-furry-error 是一个用于 React 项目的错误展示工具

它由两个部分组成：

| 项目名称 | 定位 | 使用场景 |
| :--- | :--- | :--- |
| furry-ts-error | VSCode 插件 | 基于yoavbls／pretty－ts－errors，实现开发阶段实时捕获控制台报错 |
| react-furry-error | npm 依赖 | 生产／开发运行期捕获报错 |

当你的应用在开发过程中发生错误时，它不会只显示冷冰冰的红色报错页面，而是会根据错误类型，展示 不同表情和动作的 React 兽设插画，帮助你在第一时间理解问题的大致方向。

当然，可爱并不意味着信息缺失——

非侵入式的增量开发，所有 原始错误信息（message、stack、文件位置）都会被完整保留并展示。

## 这个项目有什么不同？

- 将错误「可视化分类」
- 覆盖 React 项目中大多数常见开发错误
- 整合编辑期、构建期与运行期错误信息
- 提供统一的错误视图与上下文展示
- 不吞掉任何原始错误信息
- 不侵入 React 内部实现
- 可运行于 Vite／Webpack／CRA／Next．js（运行时错误）
react－furry－error 的目标不是隐藏错误，
而是 让你更快地理解错误。


## 注意

－此项目无法捕捉**`100%`**的错误，例如框架层错误，EsLint错误等
